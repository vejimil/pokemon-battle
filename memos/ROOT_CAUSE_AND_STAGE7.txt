[Stage 7] Singles-first battle engine expansion + battle-gimmick runtime hookup

이번 단계에서 한 일
1. 싱글 배틀 중심의 턴 엔진 구조를 강화했다.
   - 행동 선택 후 실제 턴 큐를 만들고
   - 교체를 우선 처리하고
   - 기술은 우선도 / 스피드 기준으로 정렬한 뒤 실행한다.
   - 메가진화 선택 시 그 턴의 스피드 판정에 반영되도록 projected speed 계산을 넣었다.

2. 전투 중 gimmick 상태를 side / mon 단위로 분리했다.
   - side: megaUsed / teraUsed / zUsed / dynamaxUsed
   - mon: terastallized / dynamaxed / gigantamaxed / dynamaxTurns / baseMaxHp
   - 그래서 “한 번만 사용 가능한 gimmick”과 “해당 포켓몬이 현재 변신 중인지”를 분리해서 관리한다.

3. 메가진화 / 테라스탈 / Z기술 / 다이맥스를 실제 전투 로직과 연결했다.
   - 메가진화: 종족 / 타입 / 능력치 / 특성 / 스프라이트를 전투 중 함께 변경
   - 테라스탈: 팀당 1회 제한, 타입 변경 + 기존/original 타입 STAB 유지 로직 반영
   - Z기술: 조건 검사(아이템 / 타입 / 전용 기술) 후 1회 사용, status Z 보너스 일부 반영
   - 다이맥스: 팀당 1회 제한, 3턴 지속, HP 비율 유지, Max Move 변환, Gigantamax asset은 실제 resolved asset이 있을 때만 사용

4. move runtime 변환 단계를 추가했다.
   - base move를 그대로 쓰지 않고, 실행 직전에 runtime move를 만든다.
   - 여기서
     - Tera Blast 타입/카테고리
     - 테라 저위력 기술 60 BP 보정의 기본형
     - Z기술 이름/위력 변환
     - Max Move 이름/위력 변환
     - Max Guard 처리
     를 연결했다.

5. 데미지 계산을 현재 전투 상태 기준으로 보강했다.
   - 현재 move type / category 기준으로 계산
   - 테라스탈 STAB 규칙 반영
   - 메가 후 능력치 / 타입 반영
   - Max / Z move 위력 반영
   - Focus Sash / Multiscale / Flash Fire / Levitate / Life Orb / Rocky Helmet 등 기존 구현과 연결 유지

6. HP 비율 유지형 변신을 보강했다.
   - 메가진화: 기존처럼 HP 비율 유지
   - 다이맥스: 현재 HP 비율을 유지한 채 max/current HP 확장
   - 다이맥스 종료나 교체 시에도 HP 비율을 유지한 채 원래 max HP로 복귀

7. end-of-turn 처리도 다이맥스 기준으로 보강했다.
   - burn / poison / toxic / leftovers / black sludge / sitrus berry 같은 max HP 기반 처리를
     dynamax HP가 아니라 base HP 기준으로 계산하도록 수정했다.
   - 다이맥스는 턴 종료 시 카운트가 줄고, 0이 되면 자동 해제된다.

8. UI 쪽도 전투 상태를 더 잘 드러내게 수정했다.
   - Choice panel에 Z / Dynamax 토글 추가
   - 이미 변신 중인 상태는 badge로 표시
   - 팀 카드와 필드 표시에서도 Mega / Tera / Dynamax 상태를 보이게 함
   - 다이맥스 중에는 move preview가 Max Move 이름/위력 기준으로 보이게 함

이번 단계에서 특히 조심한 점
- 에셋 파일명은 여전히 Stage 6의 resolved mapping만 사용한다.
- _mega / _gmax 식으로 파일명을 가정하지 않았다.
- Gigantamax 스프라이트도 실제 resolved Gmax asset이 있을 때만 붙이고,
  없으면 그냥 잘못된 폼을 억지로 연결하지 않고 일반 Dynamax 상태만 표시한다.

이번 단계의 현재 한계
- 이 빌드는 이제 singles 기준 battle flow가 훨씬 실제 전투 엔진에 가까워졌지만,
  아직 Showdown 전체 수준의 완전 시뮬레이터는 아니다.
- weather / terrain / field / entry hazards / PP / encore / choice lock / trick room / redirection / substitute / protect success decay 같은 고급 시스템은 아직 완전하지 않다.
- Max Move의 field effect는 아직 핵심 boost/drop 위주만 반영하고, 모든 장판/날씨를 완전하게 시뮬레이션하지는 않는다.
- Z-status의 특수 효과도 heal / clear negative boosts / stat boosts 위주로 우선 반영했다.
- doubles는 기존 기반은 남아 있지만, 이번 단계의 주력은 singles 안정화다.

다음 단계 추천
1. singles 추가 정교화
   - weather / terrain / field / hazard / choice lock / PP / struggle
   - move effect coverage 확대
   - more accurate immunity / redirection / protect interaction
2. doubles 구조 확장
   - 타겟팅 / spread / ally-interaction / double-specific UI 강화
3. gimmick 세부 심화
   - more exact Z-status effects
   - more exact Max Move field effects
   - mid-battle special forms (Aegislash, Mimikyu, etc.)
